▼
1
마크다운 코드 펜스 내 JSON 추출
LLM이 응답을 ```json ... ``` 블록으로 감쌌을 때
m = re.search(r"```(?:json)?\s*(\{[^`]*\})\s*```", text, re.DOTALL) if m: return json.loads(m.group(1)) # 성공 시 여기서 반환
2
균형 중괄호 탐색으로 완전한 JSON 객체 추출
코드 펜스 없이 JSON을 바로 출력했을 때 — depth 카운터로 중괄호 매칭
def _find_balanced(text, start): depth = 0 for i, ch in enumerate(text[start:], start): if ch == "{": depth += 1 elif ch == "}": depth -= 1 if depth == 0: return i # 매칭되는 닫는 괄호 위치 return -1 start = text.find("{") end = _find_balanced(text, start) return json.loads(text[start:end+1])
3
잘린 JSON 부분 복구 (토큰 한도 초과 대응)
응답이 도중에 끊겼을 때 — "results" 배열에서 완성된 객체만 하나씩 추출
# "results" 배열 위치 찾기 results_pos = text.find('"results"') array_start = text.find("[", results_pos) recovered = [] pos = array_start + 1 while pos < len(text): obj_start = text.find("{", pos) obj_end = _find_balanced(text, obj_start) if obj_end == -1: break # 마지막 미완성 객체 → 버림 recovered.append(json.loads(text[obj_start:obj_end+1])) pos = obj_end + 1 return {"results": recovered} # 완성된 것만 살림
⚠ logger.warning("잘린 JSON 부분 복구: N건") 로그 출력 후 계속 진행

except (ValueError, json.JSONDecodeError) as e: logger.error("Gemini 검색 JSON 파싱 실패: %s", e) logger.debug("응답 앞부분: %s", response[:300]) return [] # 다른 DB 결과는 유지됨

병렬 처리 코드
# gemini_search는 항상 추가 (dbs 목록에 없어도) all_dbs = list(dbs) + (["gemini_search"] if "gemini_search" not in dbs else []) with ThreadPoolExecutor(max_workers=len(all_dbs)) as executor: futures = {executor.submit(_fetch, db): db for db in all_dbs} for future in as_completed(futures): db, hits = future.result() for hit in hits: hit = self.cache.fetch_and_store(hit) # 로컬 캐싱 found.append(hit)
