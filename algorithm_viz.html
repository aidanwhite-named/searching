Set Cover 최적 인용문헌 매칭

허용 오차 밴드 + 조정 점수로 최소 인용으로 최대 커버리지

matcher.py ▼

조정 점수 공식

adjusted_score =
0.6 × similarity_score +
0.4 × (covered_claims / total_claims)
  • w_score=0.6 (유사도 가중치)
  • w_coverage=0.4 (커버리지 가중치)
  • 더 많은 청구항 커버 → 우선 선택

매칭 순서

  • ① 독립항 → primary ref 1개 선택
  • ② 종속항 → primary ref 재사용 우선
  • ③ primary가 커버 못하면 secondary 탐색
  • ④ 최대 인용 수: 2개 (예외 3개)

재사용 우선 원칙

  • 종속항도 부모의 primary ref가
    커버하면 → 별도 문헌 추가 안함
  • 누더기 거절 방지 효과
  • root 독립항 탐색: 부모 체인 traverse

할루시네이션 검증 (Exact Match)

LLM 추출 단락 → 원본 문서 exact match → 실패 시 재시도

hallucination_checker.py ▼

검증 로직

def _exact_match(paragraph, doc_text):
return bool(paragraph) and \
paragraph.strip() in doc_text
  • Python in 연산자로 단순 포함 검사
  • 공백 trim 후 비교

재시도 전략

  • 최대 3회 재시도
  • 1회: 일반 추출 프롬프트
  • 2~3회: "실패 알림" 포함 재추출 프롬프트
  • 최종 실패 시: paragraph_verified=False
  • 문서 텍스트 최대 4,000자 사용

no-llm 모드

  • verify_only() 메서드 사용
  • LLM 호출 없이 기존 단락만 검증
  • 비용 절감용 fallback

■ 청크 타입 분류 체계

abstract

  • 특허 요약(Abstract) 전문
  • 발명의 핵심을 압축 표현
  • 빠른 관련성 판단에 유용

summary

  • 발명의 상세한 설명 청크
  • 기술적 세부사항 포함
  • 실시예·도면 설명 포함 가능

independent_claim

  • 선행문헌의 독립 청구항
  • 핵심 기술 구성요소 명시
  • 진보성 판단 핵심 대상

claim / sub_claim

  • 선행문헌의 종속 청구항
  • 부가적 기술적 특징
  • 종속항 간 유사성 탐색

■ 벡터 검색 vs 리랭킹 비교

1단계: 벡터 검색 (Dense Retrieval)
  • • 쿼리 벡터와 저장 벡터 간 코사인 유사도 계산
  • • ANN(근사 최근접 이웃) 빠른 검색
  • • top-50개 후보 추출
  • • 단일 벡터 비교 → 속도 빠름
  • • 의미적 유사성 기반
2단계: 리랭킹 (Cross-Encoder)
  • • 쿼리 + 청크를 함께 입력으로 상호작용 모델링
  • • bge-reranker-v2-m3 cross-encoder
  • • 50개 → top-10개로 정밀 선별
  • • 속도 느리지만 정확도 높음
  • • 문맥적 관련성 기반

■ 검증 전략 비교

기본 모드 (LLM + Exact Match)

  • • 원본 텍스트와 exact match 검증
  • • 실패 시 최대 3회 재시도

  • • LLM 호출 없음 (비용 0)

  • • 빠른 배치 검증에 적합
  • • 단락 품질 보장 없음
  • • 신속한 대량 처리 목적
